## How to use

1. Paste this into a new conversation
2. Add any session-specific concerns to the **This session** section
3. Run the research
4. Bring findings back to this project for discussion and agreement before applying anything

## Base criteria

Search for better approaches to:

- Anti-sycophancy and epistemic flagging in AI behavioral prompts
- Scope control and conciseness rules
- Inline labeling systems for uncertainty, assumptions, and omissions
- Versioning and change management conventions for prompt files
- Source/dist patterns for prompt compression and distribution
- Project template conventions for Claude projects

Evaluate each finding against the existing rules in `profile-instructions.md` and `project-instructions.md`. Flag anything that:

- Does something the current rules don't cover
- Does something the current rules cover but more effectively
- Introduces a meaningful tradeoff worth discussing

Discard anything that duplicates existing coverage or trades fidelity for brevity without clear gain.
